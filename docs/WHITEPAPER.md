# 📘 Lago - 来购（社区二手租售平台）

---

## 一、产品定位与愿景

### 🎯 产品定位

Lago 是一个以 **小区为中心的社区化二手与租赁平台**，首期聚焦 **玩具与游戏机** 两大类目，打造基于邻里信任的“社区循环经济生态”。
产品形态为 **微信小程序**，通过 **Next.js Web-View 套壳前端 + Express + Prisma + PostgreSQL 后端** 架构，兼容微信生态与移动端体验。

### 💡 产品愿景

让社区成为共享经济的最小生态单元，让闲置物品在小区内实现“租、卖、换、再生”的良性循环。

---

## 二、市场机会与痛点分析

| 痛点                       | 用户视角           | Lago 对应解决方案          |
| -------------------------- | ------------------ | -------------------------- |
| 二手平台交易距离远、等待长 | 缺少即时交易       | 小区级定位，邻里即时面交   |
| 信任缺失、担心被骗         | 无真实身份体系     | 小区实名认证 + 物业背书    |
| 玩具租赁门槛高             | 需要押金、沟通繁琐 | 平台押金托管 + 智能循环柜  |
| 社区流量未被利用           | 无线上线下闭环     | 物业 + 直播 + 线下体验活动 |

## 五、AI 模块设计亮点

| 功能模块 | AI 角色                                                                      |
| -------- | ---------------------------------------------------------------------------- |
| 智能推荐 | 基于 GeoHash + 用户画像的邻里推荐引擎（可学习用户偏好、位置与互动）          |
| 智能搜索 | AI 搜索引擎（语义搜索 + 意图识别），支持模糊描述如“我想租个 Switch 一个周末” |
| 智能客服 | 优先由 AI 处理咨询、退款、押金问题，仅在用户选择时转人工客服                 |
| 智能运营 | AI 自动识别商品热度、用户投诉风险、活动效果数据并推送优化建议                |

> 🚀 **AI 是 Lago 的服务中枢，而不是外挂工具。**
> 它同时扮演搜索、客服与个性化推荐引擎三重角色。

## 六、物业合作机制与线下闭环

### 🧩 合作形式

1. **无人值守“玩具循环柜”**

   - 物业提供位置与电源
   - 平台部署智能柜，住户扫码存取
   - 平台统一管理押金与维修
   - 数据实时同步至小区频道

2. **周末线下活动 + 实时直播**

   - 平台员工定期进入小区举办“玩具回收节 / 租赁日”
   - 用户可线下交易 / 置换 / 体验
   - 同步开启小区频道直播，用户可在小程序内观看互动
   - 支持在线下单、线上支付、实时抽奖等功能

> 📺 小区频道 = 实体活动 + 内容直播 + 电商入口
> 形成了“小区 → 实体柜 → 活动 → 应用频道”的流量闭环。

## 七、虚拟号服务机制优化

| 项目       | 原设计             | 优化后方案                         |
| ---------- | ------------------ | ---------------------------------- |
| 成本承担   | 每次通话 0.99 元   | 平台可选择承担首月成本或转移至卖家 |
| 计费模式   | 每次连接           | 改为“按每个买家接入包月”模式       |
| 卖家未购买 | 直接引导加微信好友 | 平台提供防封指引与风险提示         |

> 💡 可分级收费：
> 普通卖家 9.9 元/月（限 30 位买家虚拟号通话）
> 高级卖家 19.9 元/月（不限次数）

## 八、交易链路与盈利模型升级

### 🔗 交易路径

#### 模式一：平台闭环（寄售 / 托管）

1. 卖家授权商品 → 平台验收 / 翻新
2. 平台代售 → 买家付款 → 平台收款
3. 交付后平台结算卖家
4. 平台抽佣 + 服务费（5%-15%）

#### 模式二：C2C 社区交易

1. 买卖双方直接沟通 → 面交或自提
2. 可选虚拟号通话或直接加微信
3. 平台仅记录信誉积分与活跃度

---

### 💰 盈利点重构（目标：100 小区盈亏平衡）

| 收益来源     | 模型说明                            | 占比 |
| ------------ | ----------------------------------- | ---- |
| 寄售抽佣     | 平台代售商品抽成 5%-15%             | 25%  |
| 租赁服务费   | 每笔租赁订单抽 10% + 押金息差       | 15%  |
| 广告曝光     | 小区频道直播广告、品牌合作植入      | 15%  |
| 虚拟号服务   | 分级订阅 9.9-19.9 元/月             | 10%  |
| 会员订阅     | 发布上限、曝光提升、专属客服        | 10%  |
| 物业合作分润 | 按交易额分成，激励物业参与          | 5%   |
| 数据服务     | 向玩具/亲子品牌提供社区用户画像分析 | 20%  |

> ⚖️ 盈亏平衡假设：
> 每小区月 GMV 5 万元 × 100 小区 = 500 万元
> 抽佣与服务收入 10% → 月收入 50 万元
> 平台支出（人力、活动、柜设备、推广）≈ 45 万元
> → 盈亏平衡临界点约在 **100 个活跃小区**
